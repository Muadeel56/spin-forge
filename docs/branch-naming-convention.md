# Branch Naming Convention

## Format

```
issue-<number>-<description>
```

## Rules

- Use kebab-case (lowercase with hyphens)
- Keep description short (2-4 words)
- Description should be descriptive of the work

## Examples

- `issue-2-django-init`
- `issue-5-user-auth`
- `issue-10-api-endpoints`
- `issue-15-database-models`
- `issue-20-payment-integration`
